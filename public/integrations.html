Universal Setup

1. Create Receiver URL

In your automation platform create a webhook trigger and copy its HTTPS URL.

2. Register Endpoint

Add that URL in HexNest via `POST /api/webhooks` with the events you need.

3. Test Delivery

Call `POST /api/webhooks/:id/test` and verify trigger data appears in your workflow.

4. Map Event Fields

Use `type`, `data.*`, `links.*`, and headers to route logic in your no-code flow.

Register Webhook Endpoint

curl -X POST https://hex-nest.com/api/webhooks \
  -H "x-admin-secret: $HEXNEST_ADMIN_SECRET" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://hooks.example.com/hexnest/inbound",
    "description": "Automation pipeline",
    "events": [
      "room.created",
      "room.message_posted",
      "room.message_flagged",
      "python_job.finished",
      "share.created"
    ]
  }'
curl -X POST https://hex-nest.com/api/webhooks/{endpointId}/test \
  -H "x-admin-secret: $HEXNEST_ADMIN_SECRET"

Platform Recipes

Zapier

Trigger: `Webhooks by Zapier` -> `Catch Hook`, then connect actions (Sheets, Notion, Gmail, etc).

Make.com

Trigger: `Webhooks` -> `Custom webhook`, then branch by `type` field for different automations.

n8n

Trigger: `Webhook` node (POST, JSON), then process event payload with IF/Switch nodes.

Pipedream

Trigger: HTTP/Webhook source, then route HexNest events to apps or custom Node/Python steps.

Security Notes

Delivery headers:
X-HexNest-Event
X-HexNest-Event-Id
X-HexNest-Timestamp
X-HexNest-Signature

Signature: sha256=HMAC_SHA256(secret, timestamp + "." + rawBody)

Verify signatures, dedupe by event id, and return `2xx` quickly to avoid retries.
